## 1.2 SEO Requirements Checklist

### 🔴 КРИТИЧЕСКИ ВАЖНО (обязательно для ВСЕХ страниц):

#### Basic Meta Tags (ОБЯЗАТЕЛЬНО):
```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <!-- Basic -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- SEO Meta -->
  <title>[ЗАГОЛОВОК 50-60 символов]</title>
  <meta name="description" content="[ОПИСАНИЕ 150-160 символов]">
  <meta name="keywords" content="[3-5 основных ключевых слов]">
  <meta name="author" content="[Автор или компания]">

  <!-- Canonical URL -->
  <link rel="canonical" href="https://example.com/page/">

  <!-- Favicon -->
  <link rel="icon" type="image/svg+xml" href="/favicon.svg">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">
```

#### Open Graph Tags (ОБЯЗАТЕЛЬНО):
```html
  <!-- Open Graph / Facebook -->
  <meta property="og:type" content="website">
  <meta property="og:url" content="https://example.com/">
  <meta property="og:title" content="[ЗАГОЛОВОДЛЯ СОЦСЕТЕЙ]">
  <meta property="og:description" content="[ОПИСАНИЕ ДЛЯ СОЦСЕТЕЙ]">
  <meta property="og:image" content="https://example.com/og-image.jpg">
  <meta property="og:image:width" content="1200">
  <meta property="og:image:height" content="630">
  <meta property="og:locale" content="ru_RU">
```

#### Twitter Card Tags (ОБЯЗАТЕЛЬНО):
```html
  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:url" content="https://example.com/">
  <meta name="twitter:title" content="[ЗАГОЛОВОК ДЛЯ TWITTER]">
  <meta name="twitter:description" content="[ОПИСАНИЕ ДЛЯ TWITTER]">
  <meta name="twitter:image" content="https://example.com/og-image.jpg">
```

#### Schema.org Structured Data (ОБЯЗАТЕЛЬНО):
```html
  <!-- Schema.org JSON-LD -->
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "WebSite",
    "name": "[Название сайта]",
    "url": "https://example.com/",
    "description": "[Описание сайта]",
    "publisher": {
      "@type": "Organization",
      "name": "[Название компании]",
      "url": "https://example.com/",
      "logo": {
        "@type": "ImageObject",
        "url": "https://example.com/logo.png"
      }
    }
  }
  </script>

  <!-- Organization Schema -->
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "Organization",
    "name": "[Название компании]",
    "url": "https://example.com/",
    "logo": "https://example.com/logo.png",
    "sameAs": [
      "https://twitter.com/profile",
      "https://linkedin.com/company/profile",
      "https://github.com/profile"
    ],
    "contactPoint": {
      "@type": "ContactPoint",
      "telephone": "+7-XXX-XXX-XX-XX",
      "contactType": "customer service"
    }
  }
  </script>
```

#### Geo Meta Tags (если локальный бизнес):
```html
  <!-- Geo Meta Tags -->
  <meta name="geo.region" content="[Страна, например RU]">
```

### SEO Чек-лист для контента:

```
□ Заголовок H1 (один на странице, включает keywords)
□ Подзаголовки H2-H6 (иерархия, keywords)
□ Ключевые слова в первом абзаце
□ Длина контента: минимум 1500 слов для статей
□ Изображения с alt текстом (ОБЯЗАТЕЛЬНО!)
□ Внутренние ссылки (минимум 5-10 на страницу)
□ Внешние ссылки (авторитетные источники)
□ Anchor текст с keywords (но не переспамить)
□ Мобильная версия (responsive)
□ Скорость загрузки < 3 секунд
□ URL семантический (example.com/article-title не example.com/?p=123)
```

# 🎓 ОБУЧАЮЩИЙ РАЗДЕЛ

## Частые ошибки:

### Ошибка 1: Создать потом добавить
```
❌ ПЛОХО: Создать базовый HTML → потом добавить SEO → потом добавить Schema
✅ ПРАВИЛЬНО: Создать СРАЗУ со всеми требованиями

Почему: Экономия 30-40% времени
```

### Ошибка 2: Забыть alt текст
```
❌ ПЛОХО: <img src="image.jpg">
✅ ПРАВИЛЬНО: <img src="image.jpg" alt="Описание изображения">

Почему: SEO + accessibility + AI понимание
```

### Ошибка 3: Не проверить в браузерах
```
❌ ПЛОХО: Создать в Chrome → запустить
✅ ПРАВИЛЬНО: Проверить в Chrome, Safari, Firefox, mobile

Почему: Разные браузеры рендерят по-разному
```

### Ошибка 4: Не проверить SEO
```
❌ ПЛОХО: Запустить → потом проверить SEO
✅ ПРАВИЛЬНО: Проверить SEO ДО запуска

Почему: Исправить ДО того как Google проиндексирует
```

### Ошибка 5: Не настроить monitoring
```
❌ ПЛОХО: Запустить → забыть про мониторинг
✅ ПРАВИЛЬНО: Настроить monitoring ДО запуска

Почему: Узнаешь о проблемах быстро, не от пользователей
```
