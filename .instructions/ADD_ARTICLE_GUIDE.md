## 🖼️ ПРАВИЛА РАБОТЫ С КАРТИНКАМИ (КРИТИЧНО!)

### Размеры и пропорции:
- **Рекомендуемое соотношение:** 16:9 (1920x1080 или 2048x1152)
- **Минимальная ширина:** 1920px для качественного отображения
- **Размещение:** Важный контент по ЦЕНТРУ картинки (боковые части могут обрезаться)

### Где картинки появляются:
1. **Главная страница (index.html):** Карточка с `height: 200px` и `object-fit: cover`
2. **Страница все статьи (articles.html):** Карточка с `height: 200px` и `object-fit: cover`
3. **Модальное окно статьи:** Полноразмерная с `max-height: 600px` и `object-fit: contain`

### Правило:
> **ПОДГОТОВЬ картинку ПРАВИЛЬНО с первого раза!** Если картинка обрезается → переделай с правильными пропорциями (16:9).

### ФАЗА 1: ПОДГОТОВКА ФАЙЛОВ (ЛОКАЛЬНО)

#### 1.1 Проверь исходные файлы
```bash
# В Downloads должны быть:
# - digital-monk-mechanic.md (исходник статьи)
# - article5:1.png (картинка к статье)
ls -la ~/Downloads/ | grep -E "digital|article5"
```

#### 1.2 Создай папку для новой статьи
```bash
# Определи номер новой статьи (следующий по порядку)
# Если последняя статья 5, то новая будет 6

cd /Users/nabusardar/Developer/Na8us_site/logs/articles/
mkdir article6
```

#### 1.3 Скопируй и переименуй файлы
```bash
# Из Downloads в папку статьи
cp ~/Downloads/digital-monk-mechanic.md ~/Downloads/Эд\ Доки\ и\ Фото/article6:1.png .
cp ~/Downloads/digital-monk-mechanic.md /Users/nabusardar/Developer/Na8us_site/logs/articles/article6/article6.md
cp ~/Downloads/Эд\ Доки\ и\ Фото/article6:1.png /Users/nabusardar/Developer/Na8us_site/logs/articles/article6/article6.png
```

#### 1.4 Исправь права на картинке (КРИТИЧНО!)
```bash
chmod 644 /Users/nabusardar/Developer/Na8us_site/logs/articles/article6/article6.png
```

**ПРОВЕРКА:**
```bash

# Должно быть:
# -rw-r--r-- (644) для .png
# -rw-r--r-- (644) для .md
```

### ФАЗА 2: ОБНОВЛЕНИЕ SCRIPT.JS

#### 2.1 Добавь статью в articlesData
**Файл:** `/Users/nabusardar/Developer/Na8us_site/script.js`

**Добавить после последней статьи:**
```javascript
'article6': {
    md: 'logs/articles/article6/article6.md',
    img: 'logs/articles/article6/article6.png'
}
```

**⚠️ ВАЖНО: Cache-buster для картинок!**

При открытии полной статьи картинка загружается через JavaScript. Браузеры кэшируют картинки!

**В строке ~196 есть код:**
```javascript
let html = `<img src="${article.img}?v=${cacheBuster}" ...`;
```

**Параметр `?v=${cacheBuster}`** добавляет timestamp чтобы обойти кэш браузера.

**Если обновил картинку:** Перезалей script.js на сервер! Браузеры подхватят новую версию с cache-buster.

#### 2.2 Настройки отображения картинки в модальном окне

**В script.js строка ~196:**
```javascript
style="width: 100%; max-height: 600px; object-fit: contain; ..."
```

**Параметры:**
- `max-height: 600px` — максимальная высота картинки
- `object-fit: contain` — показывает **всю** картинку без обрезки
- Если использовать `cover` — обрежет картинку!

**ПРОВЕРКА:**
```bash
grep -A 2 "'article6'" /Users/nabusardar/Developer/Na8us_site/script.js
# Должны быть пути к .md и .png
```

Примеры:

#### 3.2 Добавь карточку новой статьи (ПЕРЕД остальными)

**Позиция:** После `<div class="logs-grid">`

**Шаблон карточки:**
```html
<!-- Статья X -->
<div class="log-card" data-aos="fade-up" data-aos-delay="0" data-article="articleX">
    <div class="log-image">
        <img src="logs/articles/articleX/articleX.png" alt="Article X">
    </div>
    <div class="log-content">
        <div class="log-tags">TAGS</div>
        <h3 class="log-title">ЗАГОЛОВОК</h3>
        <p class="log-excerpt">ОПИСАНИЕ</p>
        <span class="log-link">Читать →</span>
    </div>
</div>
```

#### 3.3 Удали самую старую статью карточку

**Если добавили новую статью (например 6):**
- Убери карточку статьи 3 (она станет 4-й по свежести)
- Оставь: 6, 5, 4

#### 3.4 Обнови задержки AOS анимации

**После обновления:**
- Новая статья: `data-aos-delay="0"`
- Вторая: `data-aos-delay="100"`
- Третья: `data-aos-delay="200"`
- Кнопка "Все статьи": `data-aos-delay="300"`

### ФАЗА 7: GIT COMMIT (ОБЯЗАТЕЛЬНО!)

**Триггер:** После загрузки файлов на сервер И после проверки пользователем И после одобрения

**Что делать:**
```bash
cd /root/my-server/site
git add .
git commit -m "Добавлена статья [N]: [Заголовок]"
git push
```

**Зачем:** Бэкап нашей работы на GitHub ✅

**Правило для C.A.L.V.I.N.:**
> После успешной заливки на сервер - ВСЕГДА предлагать Git commit!
> Ждать одобрения пользователя → ТОЛЬКО потом выполнять.

**Пример сообщения:**
> "Файлы залиты на сервер, сайт работает. Сделать Git commit для бэкапа на GitHub?"
